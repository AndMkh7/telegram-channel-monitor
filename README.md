# Telegram Channel Monitor

Бот для мониторинга Telegram-каналов по ключевым словам. Отслеживает новые сообщения в указанных каналах и пересылает вам те, которые содержат заданные ключевые слова.

## Как это работает

Проект использует два клиента Telegram одновременно:

- **Userbot** — подключается от имени вашего аккаунта и слушает сообщения в каналах, на которые вы подписаны.
- **Bot** — Telegram-бот для управления настройками (каналы, ключевые слова) и отправки уведомлений.

Когда в отслеживаемом канале появляется сообщение, содержащее одно из ключевых слов, бот отправляет вам уведомление и пересылает оригинальное сообщение.

## Требования

- Python 3.8+
- Telegram API credentials (`API_ID`, `API_HASH`) — получить на [my.telegram.org](https://my.telegram.org)
- Telegram Bot Token — создать через [@BotFather](https://t.me/BotFather)

## Установка

1. **Клонируйте репозиторий:**

   ```bash
   git clone <repo-url>
   cd telegram-channel-monitor
   ```

2. **Установите зависимости:**

   ```bash
   pip install -r requirements.txt
   ```

3. **Создайте файл `.env`** в корне проекта:

   ```env
   API_ID=12345678
   API_HASH=your_api_hash_here
   BOT_TOKEN=123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
   MY_USER_ID=123456789
   ```

   | Переменная   | Описание                                                        |
   | ------------ | --------------------------------------------------------------- |
   | `API_ID`     | ID приложения из [my.telegram.org](https://my.telegram.org)     |
   | `API_HASH`   | Hash приложения из [my.telegram.org](https://my.telegram.org)   |
   | `BOT_TOKEN`  | Токен бота от [@BotFather](https://t.me/BotFather)             |
   | `MY_USER_ID` | Ваш Telegram User ID (можно узнать через [@userinfobot](https://t.me/userinfobot)) |

4. **Первый запуск** — при первом запуске Telethon попросит ввести номер телефона и код подтверждения для авторизации userbot-сессии. Сессия сохраняется в файле `user_session.session`.

## Запуск

```bash
python3 bot.py
```

**Запуск в фоне:**

```bash
python3 bot.py &
```

**Запуск в фоне с записью логов в файл:**

```bash
python3 bot.py > bot.log 2>&1 &
```

**Перезапуск:**

```bash
pkill -f bot.py; sleep 1 && python3 bot.py &
```

## Команды бота

Управление происходит через Telegram-бота в личных сообщениях. Команды доступны только владельцу (по `MY_USER_ID`).

| Команда                       | Описание                          |
| ----------------------------- | --------------------------------- |
| `/start`                      | Приветствие и список команд       |
| `/channels`                   | Показать отслеживаемые каналы     |
| `/add_channel @username`      | Добавить канал для мониторинга    |
| `/remove_channel @username`   | Удалить канал из мониторинга      |
| `/keywords`                   | Показать список ключевых слов     |
| `/add_keyword слово`          | Добавить ключевое слово           |
| `/remove_keyword слово`       | Удалить ключевое слово            |
| `/status`                     | Текущий статус мониторинга        |

## Конфигурация

Настройки каналов и ключевых слов хранятся в `config.json`:

```json
{
  "channels": ["channel_username", "another_channel"],
  "keywords": ["python", "разработчик", "flutter"]
}
```

Каналы можно указывать по `username` (без `@`) или по числовому ID. Файл обновляется автоматически при использовании команд бота.

## Безопасность

- **Не коммитьте** файлы `.env`, `*.session` в git — они содержат ваши учётные данные.
- Бот принимает команды только от пользователя с `MY_USER_ID`.
- Userbot-сессия привязана к вашему аккаунту — храните файлы сессий в безопасности.
